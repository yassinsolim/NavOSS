Support and route reports

Use GitHub Issues for reproducible bugs and route-quality reports. The structured forms ask for useful evidence while discouraging private trip data.

NavOSS support is not an emergency service. Follow posted signs and laws. Stop using an unsafe route when it is safe to do so.

Useful report details

  • App version and build number
  • iPhone model and iOS version
  • Local date and time
  • Public start area or landmark and destination
  • Expected road or maneuver and actual behavior
  • Whether Avoid highways was enabled
  • Whether Apple Maps or Google Maps differed at that same time

Do not include a home address, precise personal trip history, credential, or unrelated private screenshot.

Known limitations

The capabilities below describe the build 13 tester candidate, not a public release.

  • Calgary coverage only
  • No live traffic or traffic-aware ETA
  • No lane guidance
  • No Android release or offline routing
  • No community incident reports
  • CarPlay requires a compatible vehicle; Dashboard and cluster presentation are not included
  • Safety-camera records can change between City updates

Security

Do not disclose a vulnerability publicly. Use GitHub's private vulnerability report.
